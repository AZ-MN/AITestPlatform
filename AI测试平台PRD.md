## 二、产品定位与目标

### 2.1 产品定位

**一站式 AI 驱动全链路测试效能平台**通过大模型理解需求文档、接口、页面与缺陷日志，自动完成：用例生成 → 接口测试 → 自动化脚本生成 → 缺陷智能分析 → 质量报告输出。

### 2.2 核心目标

- 降低测试编写成本 **60%+**
- 提升回归效率 **50%+**
- 降低自动化编写门槛 **80%**
- 实现测试过程可量化、可追溯、可预警

### 2.3 业务边界

- 不替代专业测试工程师，只做**提效工具**
- 不替代现有项目管理系统，可**对接 Jira / 禅道 / TestLink**
- 不直接操作生产环境，只提供**测试环境能力**

## 五、详细功能需求（研发级）

## 5.1 平台管理后台

### 5.1.1 用户管理

- 字段：用户 ID、姓名、账号、手机号、邮箱、角色、状态、创建时间
- 功能：新增、编辑、禁用、重置密码、批量导入、导出

### 5.1.2 角色权限管理

- 自定义角色
- 菜单权限勾选
- 数据权限范围配置
- 权限预览

### 5.1.3 项目管理

- 项目名称、描述、负责人、环境配置、状态
- 项目成员添加 / 移除
- 项目数据归档 / 清理

### 5.1.4 迭代 / 版本管理

- 迭代名称、开始结束时间、关联需求
- 用例 / 缺陷 / 执行与迭代绑定

### 5.1.5 LLM 大模型配置

- 支持：本地模型 / 通义千问 / 文心一言 / 讯飞星火 / 自定义 OpenAI 格式接口
- 配置项：API Key、Endpoint、模型版本、超时时间、并发数、上下文长度
- 模型调用日志查看

### 5.1.6 系统配置

- 文件存储：本地 / MinIO/OBS
- 通知：企业微信 / 钉钉 / 邮件 / WebHook
- 安全：密码策略、登录超时、IP 白名单

## 5.2 AI 需求 & 用例中心

### 5.2.1 需求文档管理

- 支持上传：
  - TXT/MD/DOCX/PDF
  - 原型截图 / URL
- 需求解析字段：
  - 需求 ID、标题、描述、前置条件、业务规则、优先级、所属模块

### 5.2.2 AI 需求解析能力

输入：需求文本 / 文件输出：

- 功能点列表
- 业务规则提取
- 可测性校验（缺失信息提示）
- 测试点清单

### 5.2.3 AI 测试用例生成

输出标准用例结构：

- 用例标题
- 前置条件
- 测试步骤
- 预期结果
- 模块、优先级（P0/P1/P2/P3）
- 用例类型（功能 / 异常 / 边界 / 性能 / 安全）

支持：

- 重新生成
- 局部优化
- 人工修正后保存

### 5.2.4 用例库管理

- 用例树形目录（模块→子模块→用例）
- 批量导入 / 导出（Excel/CSV/ 禅道 / TestLink）
- 用例复制、移动、废弃、关联需求
- 用例执行历史记录

### 5.2.5 用例评审

- 评审单创建
- 评审意见记录
- 通过 / 驳回 / 修订
- 评审历史与对比

## 5.3 AI 接口测试中心

### 5.3.1 接口导入

支持：

- Swagger 1.x/2.x/OpenAPI 3.x
- Postman 集合文件
- 手动录入接口自动解析：
- URL、Method、Header、Query、Body、Param、Response

### 5.3.2 AI 接口用例生成

根据接口定义自动生成：

- 必填校验
- 空值 / 超长 / 特殊字符
- 类型不匹配
- 边界值
- 枚举合法 / 非法值
- 业务规则组合

### 5.3.3 接口在线调试

- 在线填参、发送、查看响应
- 环境变量：域名、Header、Token
- 全局前置脚本 / 后置脚本

### 5.3.4 接口批量执行

- 选择用例 / 模块 / 迭代执行
- 实时进度、通过率、失败列表
- 失败请求 / 响应详情留存

### 5.3.5 接口场景测试

- 多接口串联
- 参数传递（从响应提取值）
- 断言支持：状态码、body 字段、长度、包含关系

## 5.4 AI 自动化测试中心

### 5.4.1 AI 接口自动化生成

输入：接口信息 + 测试用例输出：

- Python + Requests/Pytest
- 可直接运行
- 自带断言
- 自动处理依赖与参数

### 5.4.2 AI UI 自动化生成

支持：

- 输入页面 URL / 截图 / 操作步骤
- AI 识别元素、生成操作流输出：
- Playwright 脚本
- Selenium 脚本
- 失败自动截图

### 5.4.3 脚本管理

- 在线编辑器（高亮、格式化）
- 运行 / 停止 / 日志
- 历史版本回溯

### 5.4.4 任务调度

- 立即执行 / 定时执行 / 周期执行
- 失败重试、通知策略
- 执行机器 / 容器选择

### 5.4.5 自动化报告

- 通过率、失败用例、耗时
- 失败截图 / 录像 / 日志
- 失败原因初步归类

## 5.5 AI 缺陷管理中心

### 5.5.1 缺陷提报

- 标题、环境、版本、模块、严重级别、复现概率
- 复现步骤、实际结果、预期结果
- 截图 / 日志 / 视频上传
- 指派、状态、标签

### 5.5.2 AI 缺陷能力

1. **AI 结构化**：从自然语言描述自动拆分标题、步骤、预期
2. **AI 分类**：功能 / 兼容性 / UI / 逻辑 / 性能 / 安全
3. **AI** **根因分析**：可能代码问题 / 配置 / 数据 / 需求理解
4. **AI 修复建议**：给出排查方向
5. **相似缺陷检索**：去重提示

### 5.5.3 缺陷流程

状态流转：`新建 → 确认 → 修复中 → 待验证 → 已关闭/重新打开`

### 5.5.4 批量操作

- 批量指派、批量修改状态、批量导出
- 缺陷统计筛选

## 5.6 测试数据中心

### 5.6.1 通用数据生成

- 姓名、手机号、身份证、邮箱、地址、时间、UUID、金额
- 自定义规则：长度、前缀、后缀、格式

### 5.6.2 业务数据生成

- 用户、商品、订单、支付、物流、优惠券等
- 支持自定义业务模板

### 5.6.3 数据脱敏

- 掩码：手机号、身份证、银行卡
- 替换、加密、混淆

### 5.6.4 数据预置

- 可保存为数据集
- 供用例 / 接口 / 自动化直接调用

## 5.7 质量看板 & 报告中心

### 5.7.1 项目概览看板

- 今日执行数
- 用例覆盖率
- 缺陷总数 / 未解决数
- 自动化成功率

### 5.7.2 用例统计

- 用例总数、各优先级占比
- 执行率、通过率、执行趋势
- 模块覆盖率对比

### 5.7.3 缺陷统计

- 按模块 / 人员 / 状态 / 级别分布
- 每日新增 / 关闭趋势
- 平均修复时长、平均验证时长

### 5.7.4 AI 自动报告

输入：执行数据 + 缺陷数据输出：

- 测试范围
- 执行结果
- 风险点
- 质量结论
- 后续建议支持导出 PDF/Word/ 图片

## 5.8 开放平台 & 第三方对接

### 5.8.1 Jira 对接

- 需求同步
- 缺陷双向同步
- 状态映射配置

### 5.8.2 禅道对接

- 用例导入 / 导出
- 缺陷同步
- 项目关联

### 5.8.3 Jenkins 对接

- 触发自动化任务
- 接收执行结果
- 流水线集成

### 5.8.4 WebHook

- 用例执行完成
- 自动化失败
- 缺陷新增 / 修复

### 5.8.5 平台 OpenAPI

- 项目 / 用例 / 接口 / 缺陷 / 报告开放接口
- 接口文档、签名机制、调用日志

## 六、非功能需求（研发必须落地）

### 6.1 性能

- 页面加载 ≤ 2s
- AI 用例生成 ≤ 10s / 次
- 接口批量执行支持 50 并发
- 单项目支持 10w+ 用例 / 缺陷

### 6.2 稳定性

- 服务可用性 ≥ 99.5%
- 接口超时自动重试
- 任务异常自动告警
- 数据库定期备份

### 6.3 安全

- 密码加密存储
- Token 过期机制
- 操作日志留存 ≥ 6 个月
- 文件上传查杀 / 大小限制

### 6.4 兼容性

- 浏览器：Chrome、Edge 最新版
- 接口：支持 HTTP/HTTPS
- 文档：兼容常见 Word/PDF/MD 格式

### 6.5 扩展性

- 模块化架构
- 模型可插拔
- 支持分布式执行机
- 支持容器化 K8s 部署
